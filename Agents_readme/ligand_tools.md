# 配体预处理工具使用参考文档

## 文档定位

本文档是 [ligand.md](./ligand.md) 的工具实现层补充。  
它定义各工具在标准流程（阶段1-4）中的位置、输入输出规范，以及如何判断生成文件的正确性。

---

## 一、工具与标准流程的映射关系


| 工具                      | 对应标准流程阶段           | 主要功能                                   |
| ----------------------- | ------------------ | -------------------------------------- |
| **Antechamber**         | 阶段2（电荷）+ 阶段3（参数分配） | 原子类型分配（GAFF）+ 电荷计算（AM1-BCC/RESP）+ 生成拓扑 |
| **CGenFF**              | 阶段2 + 阶段3          | 原子类型分配（CGenFF）+ 电荷分配 + 参数生成（含惩罚分）      |
| **SwissParam**          | 阶段1 + 阶段2 + 阶段3    | 加氢 + 电荷分配 + 生成 GROMACS/CHARMM 拓扑       |
| **CHARMM-GUI**          | 全流程（阶段1-4）         | 一体化处理，输出可直接用于模拟的完整体系                   |
| **acpype + Open Babel** | 阶段1 + 阶段3（降级）      | 加氢 + Gasteiger 电荷 + GROMACS 拓扑生成       |


---

## 二、各工具的详细使用规范

### 1. Antechamber (AmberTools)

#### 在流程中的定位

- 主要承担：**阶段2（电荷计算）+ 阶段3（GAFF 参数分配）**
- 适用于：已清理好的单体小分子，原子类型在 GAFF 覆盖范围内

#### 输入文件规范


| 输入   | 格式               | 规范要求                                                                                                                               |
| ---- | ---------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 配体结构 | PDB / MOL2 / SDF | • 必须是**单一分子**（无多单元、无交替构象） • 必须包含明确的原子连接信息（MOL2/SDF 的 BOND 段，或 PDB 的 CONECT 记录） • 建议**不包含氢原子**（Antechamber 会统一添加） • 文件路径和文件名不包含特殊字符 |


#### 输出文件规范


| 输出       | 格式                 | 规范要求                                                                                 |
| -------- | ------------------ | ------------------------------------------------------------------------------------ |
| 参数化后结构   | PREPIN / MOL2 / IN | • PREPIN 格式：包含原子类型、连接性、部分电荷 • MOL2 格式（`-fo mol2`）：包含 GAFF 原子类型（如 `c3`、`ca`、`n`）和部分电荷 |
| 缺失参数补充文件 | FRCMOD             | • 文本格式，包含 `MASS`、`BOND`、`ANGLE`、`DIHE` 等段 • 内容为 Antechamber 未能从 GAFF 库中匹配的参数         |
| 拓扑文件     | PRMTOP（经 tleap）    | • 与坐标文件原子顺序一致 • 力场参数完整                                                               |
| 坐标文件     | INPCRD（经 tleap）    | • 单位：埃（Å） • 原子顺序与 PRMTOP 一致                                                          |


#### 正确性判断标准


| 检查项            | 正确信号                                                        | 错误信号                                                         |
| -------------- | ----------------------------------------------------------- | ------------------------------------------------------------ |
| **原子类型完整性**    | 输出的 MOL2 文件中所有原子类型均为 GAFF 类型（如 `c3`、`ca`、`n`、`o`、`s`、`p` 等） | 出现 `du`（未定义）、`UNK`、或数字开头的原子类型                                |
| **电荷合理性**      | 部分电荷总和 = 配体净电荷（整数），各原子电荷在 -1.0 到 +1.0 之间（除少数金属/磷）           | 总和明显偏离整数，或出现 >                                               |
| **参数覆盖率**      | `parmchk2` 生成的 FRCMOD 文件**为空**或仅含少数二面角                      | FRCMOD 文件包含 `BOND`、`ANGLE` 缺失，说明 GAFF 库未覆盖该化学环境              |
| **tleap 加载状态** | `tleap` 输出 `Total atoms in file: X` 且无 `Fatal Error`        | 出现 `Could not find bond parameters` 或 `No such moleculetype` |

### 2. CGenFF (CHARMM General Force Field)

#### 在流程中的定位

- 主要承担：**阶段2 + 阶段3**
- 核心优势：提供**惩罚分（penalty score）**，是唯一自带质量自检的工具

#### 输入文件规范


| 输入   | 格式               | 规范要求                                                                              |
| ---- | ---------------- | --------------------------------------------------------------------------------- |
| 配体结构 | PDB / MOL2 / SDF | • 推荐使用**已加氢**的 MOL2 文件 • 若使用 PDB，需确保残基名与力场残基库不冲突 • 用于 CHARMM-GUI 时，可接受含蛋白的复合物 PDB |


#### 输出文件规范


| 输出   | 格式             | 规范要求                                                                                     |
| ---- | -------------- | ---------------------------------------------------------------------------------------- |
| 参数文件 | STR（CHARMM 格式） | • 包含 `MASS`、`BOND`、`ANGL`、`DIHE`、`IMPR`、`NONB` 等段 • 每个参数条目后通常有惩罚分注释（如 `! penalty= 12.3`） |
| 拓扑文件 | RTP / ITP（经转换） | • GROMACS 格式的 ITP 文件需包含 `[atomtypes]` 段 • 原子类型为 CGenFF 命名（如 `CG2R61`、`CG3C51`）           |


#### 正确性判断标准


| 检查项       | 正确信号                                     | 错误信号                         |
| --------- | ---------------------------------------- | ---------------------------- |
| **惩罚分**   | 惩罚分 < 10：高质量；10-50：可用但需验证；> 50：强烈建议人工检查  | 惩罚分 > 100 或缺失（说明参数为程序“猜测”）   |
| **原子类型**  | 所有原子类型为 CGenFF 标准命名（如 `CG`*、`NG*`、`OG*`） | 出现 `UNK` 或类型名称与力场文档不符        |
| **电荷一致性** | 原子电荷与 CGenFF 同类官能团电荷模式一致（可通过对比已知分子判断）    | 出现极端不对称电荷分布（如相邻两个碳电荷差 > 0.5） |

### 3. SwissParam

#### 在流程中的定位

- 覆盖：**阶段1（加氢） + 阶段2 + 阶段3**
- 适用：快速获得 CHARMM 力场参数，支持 SMILES 输入（可绕过 PDB 连通性问题）

#### 输入文件规范


| 输入         | 格式   | 规范要求                                                                                     |
| ---------- | ---- | ---------------------------------------------------------------------------------------- |
| SMILES 字符串 | 文本   | • 合法 SMILES，建议使用 `canonical SMILES`（可从 PubChem 获取） • 需明确质子化状态（如 `CCO` 为乙醇，`CC[O-]` 为乙醇盐） |
| MOL2 文件    | MOL2 | • 需包含 3D 坐标 • 可包含氢，若不包含则服务端自动添加                                                          |


#### 输出文件规范（通过 API 下载的 tar.gz 包）


| 输出   | 格式                        | 规范要求                                                    |
| ---- | ------------------------- | ------------------------------------------------------- |
| 参数文件 | ITP（GROMACS）/ STR（CHARMM） | • ITP 文件包含 `[atomtypes]` 段，可直接用于 GROMACS 拓扑 • 包含完整的键合参数 |
| 坐标文件 | GRO / PDB                 | • 原子顺序与 ITP 完全一致 • 已添加氢原子                               |
| 报告文件 | TXT / LOG                 | • 包含计算摘要和警告信息                                           |


#### 正确性判断标准


| 检查项         | 正确信号                                                   | 错误信号                  |
| ----------- | ------------------------------------------------------ | --------------------- |
| **原子类型**    | 原子类型为 CHARMM 风格（如 `CG2R61`）或 GAFF 风格（取决于后端）            | 出现 `UNK` 或类型名称含 `??`  |
| **ITP 完整性** | 文件包含 `[bonds]`、`[angles]`、`[dihedrals]` 段，且条目数与分子原子数匹配 | 缺失任何键合段，或条目数为 0       |
| **原子顺序**    | GRO 文件的原子顺序与 ITP 文件的 `[atoms]` 段完全一致                   | 顺序错位（可通过原子名称和残基名对齐检查） |

### 4. CHARMM-GUI Ligand Reader & Modeler

#### 在流程中的定位

- 覆盖：**全流程（阶段1-4）**
- 适用：需要一体化处理蛋白质-配体复合物的场景

#### 输入文件规范


| 输入         | 格式  | 规范要求                                           |
| ---------- | --- | ---------------------------------------------- |
| 复合物结构      | PDB | • 包含蛋白质和配体 • 配体通常以 HETATM 记录存在 • 建议去除水分子和共结晶溶剂 |
| 配体 SDF（可选） | SDF | • 若 PDB 中配体结构不完整，可单独上传 SDF 作为补充                |


#### 输出文件规范（通过 CHARMM-GUI 下载）


| 输出     | 格式              | 规范要求                                       |
| ------ | --------------- | ------------------------------------------ |
| 力场参数   | STR / ITP       | • 包含配体及蛋白质的完整力场参数 • 配体参数文件命名如 `ligand.str` |
| 模拟输入文件 | 多种格式            | • 可输出 GROMACS、NAMD、AMBER、OpenMM 等格式的完整模拟输入 |
| 体系坐标   | PDB / GRO / CRD | • 包含溶剂化、加离子后的完整体系                          |


#### 正确性判断标准


| 检查项           | 正确信号                                | 错误信号                                                      |
| ------------- | ----------------------------------- | --------------------------------------------------------- |
| **CSML 搜索结果** | 输出中显示 “Found in CSML” 并给出参数来源       | 显示 “Not found, parameters generated by CGenFF” （此时需检查惩罚分） |
| **配体识别**      | 输出文件中配体残基名唯一，未被误归为蛋白质残基             | 配体原子被分配到蛋白质链中，或残基名与标准氨基酸重名                                |
| **位置限制文件**    | 若请求生成位置限制，`posre_lig.itp` 存在且包含原子索引 | 缺失或索引与配体原子不对应                                             |

### 5. acpype + Open Babel（降级方案）

#### 在流程中的定位

- 覆盖：**阶段1（加氢）+ 阶段3（降级参数分配）**
- 定位：仅作为第一、二梯队失败时的保底方案

#### 输入文件规范


| 输入   | 格式               | 规范要求             |
| ---- | ---------------- | ---------------- |
| 配体结构 | PDB / MOL2 / SDF | 无严格要求，接受结构不完美的文件 |


#### 输出文件规范


| 输出         | 格式        | 规范要求                                                      |
| ---------- | --------- | --------------------------------------------------------- |
| GROMACS 拓扑 | TOP / ITP | • 包含 GROMACS 可读的拓扑信息 • 电荷类型为 Gasteiger 或 AM1-BCC（取决于调用方式） |
| 坐标文件       | GRO / PDB | • 已加氢 • 与拓扑原子顺序一致                                         |


#### 正确性判断标准


| 检查项        | 正确信号                                                        | 错误信号                                       |
| ---------- | ----------------------------------------------------------- | ------------------------------------------ |
| **原子类型**   | 原子类型为 GAFF 类型（若 acpype 调用 Antechamber）或通用类型（若仅用 Open Babel） | 出现 `UNK` 或类型名称为元素符号（如 `C`、`O` 而非 `c3`、`o`） |
| **电荷精度警示** | 明确标注所用电荷方法（建议使用 `-a ambe` 调用 AM1-BCC）                       | 使用 Gasteiger 电荷（默认）且未加警告                   |
| **拓扑加载**   | `gmx grompp` 可正常读取                                          | 报错 “No such moleculetype” 或原子类型未定义         |

## 三、输出文件正确性通用检查清单

无论使用何种工具，生成的文件应满足以下最低标准：

### 拓扑文件检查清单

- **原子类型存在性**：所有原子均有明确原子类型，无 `UNK`、`??`、`du` 等占位符
- **电荷守恒**：所有原子部分电荷之和 = 配体净电荷（误差 < 0.01）
- **键合参数完整**：`[bonds]` 段条目数 ≥ (原子数 - 1) × 0.8（允许少数刚性键缺失，但不应大面积缺失）
- **二面角参数**：至少包含必要的可旋转二面角参数（若配体有单键旋转）

### 坐标文件检查清单

- **原子顺序匹配**：坐标文件原子顺序与拓扑文件 `[atoms]` 段顺序完全一致
- **残基名唯一**：配体残基名（如 `LIG`）未与蛋白质标准残基名冲突
- **盒子边界合理**：若为溶剂化体系，配体距盒子边缘 ≥ 1.0 nm

## 四、阶梯式调用策略（决策树）

### 第一梯队：优先尝试


| 用户输入类型      | 首选工具                                   | 理由                   |
| ----------- | -------------------------------------- | -------------------- |
| SMILES 字符串  | **SwissParam API**                     | 绕过 PDB 连通性问题，自动加氢和电荷 |
| PDB 文件（含配体） | **CHARMM-GUI Ligand Reader & Modeler** | 一体化处理，自动 CSML 搜索     |
| MOL2 文件（单体） | **Antechamber**                        | Amber 生态标准，输出稳定      |


### 第二梯队：降级方案

当第一梯队失败时（如 Antechamber 报错“多单元”或 CHARMM-GUI 不可达）：


| 失败场景              | 降级方案                               |
| ----------------- | ---------------------------------- |
| Antechamber 连通性错误 | 使用 SwissParam（上传 MOL2）或 CGenFF 命令行 |
| 网络/服务器不可达         | 使用 acpype + Open Babel 保底          |


### 第三梯队：人工干预

若以上全部失败，Agent 应输出：

1. 失败工具的完整错误日志（如 Antechamber 的 `antechamber.err`）
2. 建议用户检查项：
  - 配体是否包含非常见元素（如 B、Si、Se）
  - 配体是否包含多单元（如二聚体、共价抑制剂）
  - PDB 文件中 CONECT 记录是否完整
3. 推荐用户手动使用 CHARMM-GUI 网页版处理

## 五、常见错误与解决方案


| 错误现象                                              | 可能原因                            | 解决方案                                    |
| ------------------------------------------------- | ------------------------------- | --------------------------------------- |
| `antechamber: Fatal Error: Cannot find bond type` | PDB 文件缺失 CONECT 记录或键连错误         | 使用 Open Babel 转换为 MOL2 后再输入             |
| `No such moleculetype` in grompp                  | ITP 文件的 `[atomtypes]` 段未在主拓扑中引入 | 确保 `#include "ligand.itp"` 位于所有其他分子类型之前 |
| Penalty score > 100 (CGenFF)                      | 配体含力场未覆盖的化学环境                   | 考虑使用量子力学计算参数，或接受降级方案                    |
| 原子类型 `du` (Antechamber)                           | GAFF 无法识别该原子化学环境                | 检查原子杂化状态，或使用 `-at gaff2` 尝试 GAFF2       |
