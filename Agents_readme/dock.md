# 分子对接标准流程参考文档

## 文档概述

本文档定义了蛋白质-配体分子对接的标准流程，适用于已完成预处理的蛋白质和配体结构。本流程聚焦于**对接前的配体与受体文件准备**、**对接计算配置**、**结果解析**以及**对接后的后续工作**，不涉及特定对接软件的具体使用说明。

分子对接是一种计算技术，用于预测小分子（配体）与目标蛋白（受体）的结合亲和力和结合构象，在计算机辅助药物设计（CADD）中发挥重要作用。对接结果的质量高度依赖于输入文件的准备质量——任何PDBQT文件的错误都可能导致后续对接计算的失败。

**对接流程概览：**
预处理后的蛋白质 (PDB) + 预处理后的配体 (MOL2/SDF/SMILES)
↓
【对接准备阶段】受体与配体格式转换（如转换为 PDBQT）
↓
【对接计算阶段】对接参数配置（活性位点盒子设置） + 对接计算执行
↓
【对接后阶段】对接结果分析（结合能、相互作用） + 复合物结构提取
↓
对接后复合物结构 → 用于后续分子动力学模拟准备

## Agent 工具接口（当前实现）

为方便 Agent 自动工作流，当前在工具注册中推荐使用以下两个函数：

### 1) get_docking_box_from_p2rank

- 功能：基于蛋白 PDB 自动运行 P2Rank，并返回 top-1 口袋的对接盒参数。
- 可选：启用 GetBox 时，会尝试用 PyMOL + GetBox 给出更精确盒子；失败可自动回退。
- 关键参数：
	- protein_pdb: 蛋白 PDB 路径
	- output_dir: P2Rank 输出目录
	- use_getbox: 是否使用 GetBox
	- extension: GetBox 扩展半径
	- fallback_to_simple: GetBox 失败是否回退
- 返回：字典，包含 center_x/center_y/center_z 和 size_x/size_y/size_z。

### 2) dock

- 功能：对接入口工具。
- 行为：
	- 手动中心模式：若传入 center_x/center_y/center_z，则直接返回对接盒摘要。
	- 自动模式：未传中心时，自动调用 P2Rank/GetBox 估算盒子并返回参数摘要。
- 注意：当前实现侧重“活性位点与盒参数生成”，对接执行可在后续步骤调用 Vina 命令完成。

### 运行环境注意事项（WSL）

1. 建议在 WSL 的 AutoMD conda 环境中运行（不要在 Windows PowerShell 直接调用 Linux 版 prank）。
2. 若启用 GetBox，需确保 PyMOL 可执行命令可用且插件路径存在。
3. 无图形环境可考虑 xvfb-run（若未安装会自动提示）。



## 流程概览

分子对接流程分为三个主要阶段：
阶段1: 对接准备 → 阶段2: 对接计算 → 阶段3: 对接后处理



## 阶段1：对接准备（文件格式转换）

### 目标

将预处理后的受体（蛋白质）和配体文件转换为对接软件可识别的格式，通常为**PDBQT格式**（AutoDock Vina标准），同时分配对接计算所需的原子类型和部分电荷。

### 为什么需要PDBQT格式？

PDBQT是AutoDock系列软件特有的文件格式，在标准PDB格式基础上增加了**电荷（Q）** 和**原子类型（T）** 两列关键信息。可以简单理解为：PDBQT = PDB + 部分电荷 + AutoDock原子类型。原子类型分配错误是最常见的PDBQT文件问题之一，会直接影响对接结果的准确性。

### 输入文件规范

| 文件类型 | 格式 | 规范要求 | 来源 |
|---------|------|----------|------|
| 受体（蛋白质）文件 | PDB | • 已完成预处理（参见 `protein_param.md`）<br>• 包含完整氢原子<br>• 标准氨基酸残基完整<br>• 无配体、水分子、非标准残基（除非需保留） | `prepare_protein()` 输出 |
| 配体文件 | MOL2 / SDF / PDB / SMILES | • 已完成预处理（参见 `ligand_param.md`）<br>• 包含完整氢原子<br>• 包含部分电荷<br>• 原子类型已分配（如GAFF原子类型）<br>• MOL2文件需包含完整的原子连接信息 | `prepare_ligand()` 输出 |

**关键约束**：

- **配体净电荷必须正确**：对接软件需要知道配体的总电荷来正确计算静电相互作用。净电荷可通过SMILES计算或从预处理阶段获取。
- **预处理文件是必需的**：对接软件通常不执行加氢和电荷计算，必须使用已预处理的输入文件。
- **一个对接作业通常包含一个受体和一个或多个配体**。

### 输出文件规范

| 输出类型 | 格式 | 规范要求 |
|---------|------|----------|
| 受体PDBQT文件 | PDBQT | • 包含完整的原子坐标<br>• 包含部分电荷列（第11列）<br>• 包含AutoDock原子类型列（第12列）<br>• 无缺失原子类型 |
| 配体PDBQT文件 | PDBQT | • 格式同上<br>• 包含配体的部分电荷和原子类型 |

### PDBQT文件格式详解

一个标准的PDBQT文件包含以下13列信息：

| 列号 | 内容 | 说明 |
|------|------|------|
| 1 | 记录类型 | ATOM 或 HETATM |
| 2 | 原子序号 | 原子的唯一编号 |
| 3 | 原子名称 | 原子的化学名称 |
| 4 | 残基名称 | 所属氨基酸/配体残基名 |
| 5 | 链标识符 | 蛋白质链标识（受体）或空白（配体） |
| 6 | 残基序号 | 残基在序列中的位置 |
| 7-9 | X/Y/Z坐标 | 原子在空间中的位置（Å） |
| 10 | 占有率 | 原子在结构中的占有率 |
| 11 | 温度因子 | 原子的热振动参数 |
| 12 | **部分电荷（Q）** | 原子的电荷值，用于静电计算 |
| 13 | **原子类型（T）** | AutoDock标准原子类型（如C、N、O、H等） |

**标准原子类型**：AutoDock Vina使用的原子类型包括：C（碳）、N（氮）、O（氧）、H（氢）、S（硫）、P（磷）、F（氟）、Cl（氯）、Br（溴）、I（碘）、Mg（镁）、Mn（锰）、Zn（锌）、Ca（钙）、Fe（铁）、Na（钠）、K（钾）等。注意大小写敏感。

### 关键验证点

生成PDBQT文件后，**务必进行以下检查**：

- [ ] 使用文本编辑器打开文件，确认最后两列（部分电荷和原子类型）存在且为非空值
- [ ] 原子类型为AutoDock标准类型，大小写正确
- [ ] 电荷值为有效数值（通常在-2到+2之间）
- [ ] 无重复的原子或残基
- [ ] 受体文件中无配体残留（除非进行柔性对接）
- [ ] 配体文件的原子顺序与原始MOL2文件一致


## 阶段2：对接计算

### 目标

配置对接参数（重点是定义蛋白质活性位点区域）并执行对接计算，生成配体在受体结合位点内的预测构象和结合亲和力。

### 输入文件规范

| 文件类型 | 格式 | 规范要求 |
|---------|------|----------|
| 受体PDBQT | PDBQT | 来自阶段1，已分配原子类型和电荷 |
| 配体PDBQT | PDBQT | 来自阶段1，已分配原子类型和电荷 |

### 对接参数配置

#### 2.1 活性位点定义（Grid Box / Binding Site）

活性位点定义了配体可能结合的区域，是对接计算中最关键的参数。

**常用定义方式**：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **基于已知配体** | 使用共晶结构中的配体坐标定义盒子中心 | 有共晶结构的同源蛋白 |
| **基于催化残基** | 以已知活性位点的关键残基（如催化三联体）为中心 | 已知活性位点但无共晶配体 |
| **基于结合腔预测** | 使用P2Rank、Fpocket等工具预测可能的结合口袋 | 全新靶点，无已知活性位点信息 |

**盒子参数规范**（以AutoDock Vina为例）：

| 参数 | 说明 | 默认/推荐值 |
|------|------|------------|
| center_x, center_y, center_z | 盒子中心坐标（Å） | 基于配体或活性残基计算 |
| size_x, size_y, size_z | 盒子在各维度的半长（Å） | 通常20-30 Å，确保覆盖整个活性位点 |

**易错点**：

- **盒子大小设置不当是导致对接分数不合理的最常见原因之一**。盒子过小会遗漏真实结合位点，盒子过大则会增加搜索空间、降低计算效率和准确性。
- 盒子应**完全包含目标活性位点**，同时适度延伸以覆盖可能的配体构象空间。

#### 2.2 对接模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **刚性对接** | 受体视为刚性，仅配体构象柔性 | 快速虚拟筛选，受体结构已知且活性位点刚性较强 |
| **半柔性对接** | 受体刚性，配体柔性（可旋转键可动） | 标准分子对接（最常用模式） |
| **柔性对接** | 受体部分残基侧链可动，配体完全柔性 | 诱导契合效应显著的情况 |

对于大多数场景，**半柔性对接**（受体刚性+配体柔性）是平衡计算效率与准确性的最佳选择。

#### 2.3 对接参数文件示例（AutoDock Vina配置）
```
receptor = receptor.pdbqt
ligand = ligand.pdbqt
center_x = 12.5
center_y = 15.2
center_z = 8.3
size_x = 25
size_y = 25
size_z = 25
exhaustiveness = 8
num_modes = 9
energy_range = 3
```


### 输出文件规范

| 输出类型 | 格式 | 规范要求 |
|---------|------|----------|
| 对接结果文件 | PDBQT | • 包含所有预测构象（poses）<br>• 每个pose包含结合亲和力评分<br>• 原子坐标完整 |
| 对接日志 | TXT/LOG | • 包含对接参数、运行时间、各pose评分<br>• 可选包含RMSD值 |
| 复合物结构（可选） | PDB | • 对接后的受体-配体复合物结构<br>• 可用于后续MD模拟准备 |

**对接结果文件通常包含**：

- 每个预测构象的结合亲和力（AutoDock Vina单位为kcal/mol，数值越负结合越强）
- 各构象的原子坐标（可从文件中提取用于后续分析）
- 对接参数汇总信息

### 关键验证点

- [ ] 对接计算成功完成，无错误退出
- [ ] 输出PDBQT文件包含预期数量的pose（如 `num_modes=9` 应输出9个构象）
- [ ] 各pose的结合亲和力值在合理范围内（通常 -15 到 0 kcal/mol）
- [ ] 各pose之间的RMSD差异合理（太相似说明采样不足，差异太大说明存在多种结合模式）


## 阶段3：对接后处理

### 目标

分析对接结果，选择最佳构象，提取复合物结构用于后续分子动力学模拟准备。

### 输入文件规范

| 文件类型 | 格式 | 规范要求 |
|---------|------|----------|
| 对接结果文件 | PDBQT / PDB | 来自阶段2的输出 |
| 原始受体结构 | PDB | 对接前使用的受体PDB文件 |

### 3.1 结果分析与构象选择

**结合亲和力**：比较各pose的评分，通常选择**评分最负（结合最强）的构象**作为最佳候选。但需注意：

- 评分差异很小（<0.5 kcal/mol）时，多个构象可能都是合理的结合模式
- 评分最好的构象不一定在结构上最合理，需结合相互作用分析

**聚类分析**：根据构象的RMSD对poses进行聚类，选择每个聚类中评分最好的代表构象。

**相互作用分析**：

- 氢键：检查配体与受体关键残基之间是否存在氢键
- 疏水相互作用：检查疏水区域是否匹配
- π-π堆积：芳香环之间的堆积相互作用
- 静电相互作用：电荷互补性

### 3.2 复合物结构提取

对接后获得的最佳构象是配体在受体中的预测结合模式，该复合物结构可直接用于**分子动力学模拟的初始结构**。

**提取方法**：

1. 从对接结果PDBQT文件中提取配体坐标
2. 将配体坐标与原始受体PDB文件合并
3. 生成完整的蛋白质-配体复合物PDB文件

**输出文件规范**：

| 输出类型 | 格式 | 规范要求 |
|---------|------|----------|
| 对接后复合物 | PDB | • 包含受体蛋白的完整原子<br>• 包含配体的完整原子<br>• 配体残基名唯一（如LIG）<br>• 无原子重叠或冲突 |
| 最佳构象配体 | PDB / MOL2 | • 单独保存最佳构象的配体结构<br>• 包含正确的原子类型和电荷信息 |

### 3.3 对接后复合物的质量验证

在将对接后复合物用于MD模拟之前，需验证：

- [ ] **原子冲突检查**：配体与受体之间无异常原子重叠（最近距离≥1.5 Å）
- [ ] **几何合理性**：配体构象的键长、键角符合化学规则
- [ ] **结合模式合理性**：配体朝向与已知活性位点信息一致（如有参考结构）
- [ ] **质子化状态一致性**：对接后配体的质子化状态与预处理时一致

### 后续工作

对接后获得的复合物结构可作为**分子动力学模拟的初始输入**。在开始MD模拟前，通常需要：

1. 使用 `prepare_protein()` 或类似工具对复合物进行最终预处理
2. 将复合物作为蛋白质输入，配体通过 `keep_nonstandard_residues` 参数保留
3. 单独对配体进行力场参数分配（与对接前预处理相同）
4. 合并蛋白质和配体的拓扑与坐标，构建完整的模拟体系

## 常见问题排查（易错点汇总）

### 问题1：缺少原子类型列（T列）

- **现象**：Vina报错 "An internal error occurred in parse_pdbqt.cpp"
- **原因**：使用旧版脚本生成PDBQT文件，实际输出为PDBQ格式（缺少原子类型列）
- **解决方案**：使用新版准备脚本（`prepare_ligand4.py` 和 `prepare_receptor4.py`）；生成后用文本编辑器检查最后两列是否为有效内容

### 问题2：受体文件格式不匹配

- **现象**：对接计算无法正常启动
- **原因**：受体文件为PDBQS格式（含溶剂化参数但不含原子类型），Vina要求PDBQT格式
- **解决方案**：统一使用 `prepare_receptor4.py` 准备受体文件

### 问题3：原子类型定义不规范

- **现象**：报错 "Atom type XXX is not a valid AutoDock type"
- **原因**：原子类型不符合AutoDock标准，或大小写敏感错误
- **解决方案**：检查并修正原子类型定义，确保使用AutoDock标准原子类型（C、N、O、H、S等）；注意原子类型大小写敏感[reference:20]

### 问题4：缺少氢原子

- **现象**：对接结果不准确或氢键分析缺失
- **原因**：输入文件未在预处理阶段添加氢原子
- **解决方案**：确保在对接准备前已完成加氢操作（参见 `ligand_param.md` 和 `protein_param.md`）

### 问题5：电荷分配错误

- **现象**：分子间相互作用计算错误，对接结果不合理
- **原因**：配体净电荷不正确，或使用了错误的力场参数文件
- **解决方案**：使用正确的力场参数（如AutoDock4使用 `AD4_parameters.dat`，含锌蛋白使用 `AD4Zn.dat`）；通过SMILES验证配体净电荷

### 问题6：盒子大小或位置设置不当

- **现象**：对接分数不合理，配体未结合到预期活性位点
- **原因**：盒子太小（遗漏真实结合位点）或中心位置偏离
- **解决方案**：基于共晶配体坐标（如有）或关键活性残基坐标设置盒子中心；使用结合腔预测工具验证活性位点位置

### 问题7：输出构象异常（配体游离在活性位点外）

- **现象**：所有预测构象中配体均未结合在蛋白质表面
- **原因**：盒子未覆盖活性位点，或蛋白质结构异常（如结合位点被非标准残基/水分子占据）
- **解决方案**：检查蛋白质预处理阶段是否正确移除非标准残基和水分子；验证活性位点区域结构完整性

### 问题8：不同构象之间RMSD差异过小

- **现象**：`num_modes=9` 但9个构象几乎完全相同
- **原因**：对接采样不足（exhaustiveness过低）或配体柔性受限
- **解决方案**：增加 `exhaustiveness` 参数值（如从8增加到16或32）；检查配体可旋转键数量是否正常


## 对接后复合物质量检查清单

在将对接后复合物用于分子动力学模拟之前，需完成以下检查：

### 原子与键合检查

- [ ] **原子冲突检查**：配体与受体之间最近原子距离 ≥ 1.5 Å
- [ ] **键合完整性**：配体内部的键连接完整（无断裂）
- [ ] **氢原子完整**：对接后复合物中的配体氢原子完整（可重新加氢确保一致性）
- [ ] **残基名唯一**：配体残基名（如 LIG）未与蛋白质标准残基冲突

### 几何合理性检查

- [ ] **构象合理性**：配体键长、键角符合化学规则（无异常拉伸）
- [ ] **环构象**：芳香环和脂肪环的构象合理（非平面环无不合理扭曲）
- [ ] **立体化学**：手性中心构型正确（与输入配体一致）

### 相互作用检查

- [ ] **活性位点定位**：配体位于蛋白质活性位点区域内（非表面随机吸附）
- [ ] **氢键网络**：存在至少一个合理的氢键相互作用（有助于验证对接合理性）
- [ ] **疏水匹配**：配体疏水区域与受体疏水口袋匹配

### 一致性检查

- [ ] **原子顺序**：对接后复合物中蛋白质原子顺序与预处理后一致
- [ ] **质子化状态**：配体质子化状态与预处理阶段一致（pH条件匹配）
- [ ] **电荷守恒**：配体部分电荷之和等于净电荷（误差 < 0.01）
